# AI Interview Simulator

AI Interview Simulator 是一个面向互联网大厂技术实习与复试准备的 AI 模拟面试官。它不是题库刷题工具，而是围绕候选人的简历和项目经历进行连续追问，并把一次面试暴露出的问题反向转化为简历修改建议、范式回答和下一轮训练方向。

## 项目简介

本项目面向准备技术岗实习面试的学生，解决几个常见问题：

- 缺少能高频对练的“面试官”；
- 平常的准备太宽松，不能适应高压思考环境
- 面试后不知道自己具体卡在哪里，也不知道项目经历应该如何改写。

产品支持两种训练模式：

- **完整模拟面试**：上传 PDF 或粘贴脱敏简历，填写目标公司、岗位和面试时长，系统解析简历后生成一轮技术面试。
- **项目强化模式**：粘贴一个重点项目经历，选择提问方向和 1-3 轮强化训练，系统围绕该项目进行高强度项目深挖。

## 核心功能

- **简历驱动提问**：支持 PDF 简历解析或直接粘贴脱敏简历文本，大模型提取项目经历、技能关键词和风险点。
- **项目强化训练**：针对单个重点项目，按架构设计、技术难点、性能优化、数据库/缓存、工程协作、项目复盘或自定义方向生成问题。
- **连续追问**：用户回答后，大模型判断是否追问，并给出追问问题和“面试官思考”提示。
- **计时压力**：每题 3 分钟倒计时，回答耗时会传给后端，影响追问和总结。
- **复盘总结**：结束后展示问答记录、评分、暴露问题、简历修改建议和面试练习建议。
- **项目简历改写**：项目强化模式下，根据问答过程生成“原句/改句/修改理由”。
- **范式回答**：总结页可针对某个问题按需生成简短范式回答，帮助用户学习更好的回答结构。
- **进度回溯**：总结页可回到任意主问题检查点，清除后续记录并重新作答。
- **规则兜底**：模型未配置、超时或返回格式异常时，系统仍可用规则路径完成基本演示。

## 运行方式

### 1. 安装依赖

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 配置大模型（可选）

不配置模型也可以使用规则兜底完成基本演示。如需接入 OpenAI 兼容接口，复制 `.env.example` 为 `.env` 并填写：

```text
LLM_API_KEY=
LLM_BASE_URL=
LLM_MODEL=
```

或者在app\core\config.py中进行更改

### 3. 启动服务

```bash
uvicorn app.main:app --reload
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 技术栈说明

- **后端框架**：FastAPI
- **页面模板**：Jinja2
- **前端实现**：原生 HTML / CSS / JavaScript
- **PDF 解析**：pypdf
- **数据模型**：Pydantic
- **大模型接口**：OpenAI 兼容 Chat Completions API
- **会话存储**：当前使用进程内内存字典，适合 Demo；生产环境可替换为 SQLite / Redis
- **日志**：Python logging，写入 `logs/app.log`

不同大模型调用的等待时间：

- 简历分析、面试规划、最终总结：最长等待 120 秒；
- 正常问答中的追问判断：最长等待 60 秒。

## 项目结构

```text
app/
  api/          # HTTP 路由
  core/         # 配置
  schemas/      # 请求与响应模型
  services/     # 简历解析、Prompt、面试流程、项目强化
  static/       # 前端静态资源
  templates/    # 页面模板
samples/        # 手动体验样例
logs/           # 运行日志
```

## 调试与样例

- `logs/app.log`：业务调试日志，记录用户输入摘要、模型调用摘要、追问判断和总结来源。每次服务重启会清空旧日志。
- `samples/`：手动体验样例，包含后端、前端、算法方向的脱敏简历，以及不同质量的回答样例。

## 后续优化方向

1. 增加 OCR 服务作为 PDF 文本抽取失败时的增强能力。
2. 增加语音输入和语音播放，加强真实面试临场感。
3. 增加历史报告导出，方便用户对比多轮训练效果。
