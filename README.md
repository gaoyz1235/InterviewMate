# AI Interview Simulator

面向互联网大厂技术实习与复试准备的 AI 模拟面试官。产品重点不是题库刷题，而是围绕候选人的简历和项目经历进行连续追问，并在结束后反向给出简历修改建议和面试练习建议。

当前支持两种模式：

- **完整模拟面试**：上传 PDF 或粘贴脱敏简历，填写目标公司、岗位和面试时长，系统解析简历后生成一轮技术面试计划。
- **项目强化模式**：用户粘贴一个重点项目经历，选择提问方向和 3-5 轮强化训练，系统围绕该项目进行高强度项目深挖。

## 技术栈

- Python 3.11+
- FastAPI
- Jinja2
- pypdf
- 原生 HTML/CSS/JavaScript
- OpenAI 兼容大模型接口

## 本地运行

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

浏览器打开 `http://127.0.0.1:8000`。

## 核心特点

- **简历驱动**：支持 PDF 简历解析或直接粘贴脱敏简历文本。
- **LLM 优先解析**：简历分析、面试规划、追问判断、最终总结都优先调用大模型。
- **规则兜底**：模型未配置、超时或返回格式异常时，系统仍可用规则路径完成演示。
- **连续追问**：用户回答后，大模型判断是否继续追问，并同时生成追问问题。
- **项目强化**：可单独输入一个项目经历，选择架构设计、技术难点、性能优化、数据库/缓存、工程协作、项目复盘或自定义方向进行 3-5 轮训练。
- **计时压力**：每题 3 分钟倒计时，回答耗时会传给后端并影响追问与总结。
- **复盘输出**：结束后展示问答记录、评分、暴露问题、简历修改建议和下一步练习建议。
- **调试友好**：运行日志写入 `logs/app.log`，每次服务重启会清空并重新记录。

## 两种使用模式

### 完整模拟面试

适合想模拟一轮技术一面的场景：

- 上传 PDF 或粘贴脱敏简历文本。
- 填写目标公司、岗位和面试时长。
- 系统提取项目经历、技能关键词和简历风险点。
- 大模型生成面试计划，题型包括项目深挖、技术基础、岗位匹配、行为动机。
- 对话页支持题型显示、3 分钟倒计时、回答耗时记录。
- 结束页展示问答记录、四维评分、暴露问题、简历修改建议和练习建议。

### 项目强化模式

适合重点打磨简历中的某一个项目：

- 粘贴一个具体项目经历。
- 选择提问方向：架构设计、技术难点、性能优化、数据库/缓存、工程协作、项目复盘或自定义。
- 选择强化轮数：3、4 或 5 轮。
- 系统围绕该项目生成主问题，并在每轮根据回答继续追问。
- 总结更聚焦项目表达、技术深度、抗追问能力和简历呈现。

## LLM 配置

不配置模型也可以使用规则兜底完整演示。如需接入 OpenAI 兼容接口，复制 `.env.example` 为 `.env` 并填写：

```text
LLM_API_KEY=
LLM_BASE_URL=
LLM_MODEL=
```

不同调用场景使用不同等待时间：

- 简历分析、面试规划、最终总结：最长等待 120 秒。
- 正常问答中的追问判断：最长等待 60 秒。

## 日志与样例

- `logs/app.log`：业务调试日志，记录用户输入摘要、模型调用摘要、追问判断和总结来源。每次服务重启会清空旧日志。
- `samples/`：手动体验样例，包含后端、前端、算法方向的脱敏简历，以及不同质量的回答样例。

## 项目结构

```text
app/
  api/          # HTTP 路由
  core/         # 配置
  schemas/      # 请求与响应模型
  services/     # 简历解析、Prompt、面试流程、项目强化
  static/       # 前端静态资源
  templates/    # 页面模板
samples/        # 手动体验样例
logs/           # 运行日志
tests/          # 测试
docs/           # 产品文档、Memo、部署说明
```

## 后续优化

1. 增加 SQLite 或 Redis 持久化，避免服务重启丢失会话。
2. 增加 OCR 服务作为 PDF 文本抽取失败时的增强能力。
3. 增加语音输入/语音播放，加强真实面试临场感。
4. 增加历史报告导出，方便用户对比多轮训练效果。
